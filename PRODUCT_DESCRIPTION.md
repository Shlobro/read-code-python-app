# Product Description

ReadCode is a learning app focused on understanding code rather than writing it.

## Core Experience
- Users choose a problem type first.
- Users then choose a difficulty level.
- Users see a list of problems for that type and difficulty.
- All interactions are tap-based; users should never need to type answers.
- Most problem types are multiple choice. The Arrange the Code type uses a tap-to-order interaction.
- The list should show whether a problem is complete or incomplete.
- Users should also be able to jump into a random problem from the current filtered list.

## Current Scope
- All problems are Python problems.
- The data model should still support language as a first-class property so the app can later add an app-wide default and per-problem language selection.

## Problem Types
- Find the Bug
- What Is the Output?
- What Does This Code Do?
- Fill in the Blank
- Arrange the Code
- Complexity Check
- Variable Trace
- Match the Output

## Difficulty Levels
- Easy
- Medium
- Hard

## Product Priorities
- Clear menu and submenu navigation
- Fast, no-keyboard interaction
- Good readability for code snippets
- Easy expansion of the problem bank
